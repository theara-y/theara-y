        Dynamic functions and classes
        =============================
        def __init__(self, dynamic_name):
            self.dynamic_name = dynamic_name
            def my_fn(self):
                print(self.dynamic_name)

        bound_method = my_fn.__get__(self, self.__class__)
        setattr(self, self.dynamic_name, bound_method)

        Generators
        ==========
        - like using an iterator with next
        - saves memory, data too big to hold in memory
        - generators can yield infinitely

        def generateNumber(n):
            current = 0
            while current < n:
                yield current
                current += 1

        iterator = generateNumber(100)
        next(iterator) // next yielded number

        __repr__
        ========
        - Can overwrite this method on classes to change how to display an object
        // app.py
        class User:
            def __init__(self):
                self.name = 'Alice'

            def __repr__(self):
                return f'User: {self.alice}'


        Progamming Paradigms
        ====================
        - Procedural
            - Linear, step by step tasks
            - tasks encapsulated into modular procedures such as functions
            - procedure, routine, subroutine
        - Object-oriented
            - Concept of objects
            - attributes, states, properties
            - methods, behavior
        - Functional
            - pure functions, avoidance of side effects, immutable data

        OOP
        ===
        - classes are blueprints used to create instances of objects
        - classes as custom data types or data structures
        - inheritance
            - child inherits parent, subclass extends superclass, sharing base 
            - super().__init__()
        - everything in Python is an object
            - except variables
